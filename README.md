# DOCX Templater Plugin for Dify / Плагин DOCX Templater для Dify

A Dify tool plugin that dynamically generates `.docx` (Microsoft Word) files from `.docx` templates and JSON context data using the [python-docx-template](https://github.com/elapouya/python-docx-template) framework.

Плагин-инструмент для Dify, который динамически генерирует файлы `.docx` (Microsoft Word) из шаблонов `.docx` и JSON-данных с использованием фреймворка [python-docx-template](https://github.com/elapouya/python-docx-template).

---

## Features / Возможности

- **EN:** Upload any `.docx` file with Jinja2 placeholders and render it with JSON data
- **RU:** Загружайте любой файл `.docx` с плейсхолдерами Jinja2 и рендерьте его с помощью JSON-данных
- **EN:** Support for variables, loops, conditionals, and table row loops
- **RU:** Поддержка переменных, циклов, условий и циклов по строкам таблиц
- **EN:** Output is a fully formatted Word document ready for download
- **RU:** Результат — полностью отформатированный документ Word, готовый к скачиванию

## Installation & Deployment / Установка и развёртывание

### Prerequisites / Предварительные требования

- Dify instance (self-hosted or cloud)
- Python 3.12+ (for local development and testing)
- Dify Plugin CLI: `pip install dify-plugin`

### Step 1: Package the plugin / Шаг 1: Сборка пакета

```bash
# Clone or download this repository
git clone <repo-url>
cd dify-plugin-gelonsoft-docx-templater

# Package into .difypkg file
dify-plugin package ./
```

### Step 2: Install in Dify / Шаг 2: Установка в Dify

1. Open your Dify dashboard → **Plugins** → **Install**
2. Upload the generated `.difypkg` file
3. The plugin will appear in your tool list as **DOCX Templater**

1. Откройте панель Dify → **Плагины** → **Установить**
2. Загрузите сгенерированный файл `.difypkg`
3. Плагин появится в списке инструментов под названием **DOCX Templater**

## How to Use / Как использовать

### Tool: Render Template / Инструмент: Рендер шаблона

#### Parameters / Параметры

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `template` | file | Yes | `.docx` template file with Jinja2 placeholders |
| `context` | string | Yes | JSON string with data to fill the template |
| `filename` | string | No | Output filename without `.docx` (default: `output`) |

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `template` | файл | Да | Файл шаблона `.docx` с плейсхолдерами Jinja2 |
| `context` | строка | Да | JSON-строка с данными для заполнения шаблона |
| `filename` | строка | Нет | Имя выходного файла без `.docx` (по умолчанию: `output`) |

#### Workflow / Рабочий процесс

1. **Create a template** in Microsoft Word, LibreOffice Writer, or any .docx editor
2. **Insert Jinja2 placeholders** where dynamic data should appear
3. **Upload the template** to the Dify tool node
4. **Provide JSON context** as a string (can be generated by LLM in the workflow)
5. **Run the workflow** — the output node will contain the generated `.docx` file

1. **Создайте шаблон** в Microsoft Word, LibreOffice Writer или любом редакторе .docx
2. **Вставьте плейсхолдеры Jinja2** в места, где должны появляться динамические данные
3. **Загрузите шаблон** в узел инструмента Dify
4. **Передайте JSON-контекст** в виде строки (может быть сгенерирован LLM в рабочем процессе)
5. **Запустите рабочий процесс** — выходной узел будет содержать сгенерированный файл `.docx`

## Template Syntax Reference / Справочник по синтаксису шаблонов

### Basic Variables / Простые переменные

```
{{ company_name }}
{{ date }}
{{ total_amount }}
```

Insert any text value. Use spaces around variable names for best compatibility.

Вставляет любое текстовое значение. Для лучшей совместимости используйте пробелы вокруг имен переменных.

### Loops / Циклы

```
{% for item in items %}
  {{ loop.index }}. {{ item.name }} — {{ item.price }} ₽
{% endfor %}
```

Iterate over lists. `loop.index` gives the 1-based counter.

Итерирует по спискам. `loop.index` даёт счётчик, начинающийся с 1.

### Conditionals / Условия

```
{% if discount_percent > 0 %}
  Скидка: {{ discount_percent }}%
{% else %}
  Скидка не применена
{% endif %}
```

Show or hide content based on conditions.

Показывает или скрывает контент на основе условий.

### Table Row Loops / Циклы по строкам таблицы

In the **first cell** of the table row you want to repeat:

В **первой ячейке** строки таблицы, которую нужно повторить:

| № | Product | Price |
|---|---------|-------|
| {%tr for item in items %} | | |
| {{ loop.index }} | {{ item.name }} | {{ item.price }} |
| {%tr endfor %} | | |

This repeats the entire row for each item in the list.

Это повторяет всю строку для каждого элемента списка.

### Rich Text (Optional) / Форматированный текст (опционально)

Use `{{r variable }}` for `RichText` objects with bold, colors, hyperlinks.

Используйте `{{r variable }}` для объектов `RichText` с жирным шрифтом, цветами, гиперссылками.

## Tips / Советы

1. **Always use spaces** around Jinja2 variables: `{{ name }}` not `{{name}}`
2. **Table row loops**: Place `{%tr for ... %}` in the **first cell** of the row to repeat
3. **JSON context**: Ensure numbers used in comparisons are actual numbers, not strings with spaces
4. **Russian text**: Full Unicode support — Cyrillic works in templates, JSON, and output
5. **File uploads in Dify**: The template file is passed to the tool with a `.path` attribute pointing to the uploaded file

1. **Всегда используйте пробелы** вокруг переменных Jinja2: `{{ name }}`, а не `{{name}}`
2. **Циклы по строкам таблицы**: Помещайте `{%tr for ... %}` в **первую ячейку** повторяемой строки
3. **JSON-контекст**: Убедитесь, что числа для сравнений — настоящие числа, а не строки с пробелами
4. **Русский текст**: Полная поддержка Unicode — кириллица работает в шаблонах, JSON и выходных документах
5. **Загрузка файлов в Dify**: Файл шаблона передаётся инструменту с атрибутом `.path`, указывающим на загруженный файл
